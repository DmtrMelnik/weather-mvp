# Погода — несколько источников

## Идея проекта

Веб-приложение, которое в одном месте показывает погоду из нескольких источников и моделей. Пользователь вводит город или координаты и получает:

- **Текущую погоду** от разных провайдеров (Open-Meteo, OpenWeatherMap, DWD ICON) в виде карточек для сравнения.
- **Прогноз на неделю** с детальным ветром по дням.
- **Ветер по высотам** — скорость и направление на 10, 80, 120, 180 м и по уровням давления (~110 м, ~320 м, … до ~1000 м), в том числе для планирования полётов и ветроэнергетики.

В приоритете — использование модели **DWD ICON** (в том числе ICON-D2 для Центральной Европы) как одной из самых точных для региона Беларуси и Восточной Европы.

---

## Какие проблемы закрывает

- **Разброс данных** — не нужно открывать несколько сайтов погоды; всё в одном интерфейсе.
- **Один город — одна заглушка** — вместо фиксированного «Берлина» работает любой город и координаты за счёт геокодинга.
- **Нет прогноза на неделю** — появляется почасовой и дневной прогноз на 7–10 дней с акцентом на ветер.
- **Нет ветра по высотам** — отображается ветер на нескольких высотах (10–180 м и по уровням давления до ~1000 м) на основе DWD ICON.
- **Нет «правильной» модели для региона** — в качестве одного из источников используется DWD ICON (включая ICON Europe / ICON-D2 где доступно).

---

## Где должно работать

- **Десктоп** — основные сценарии (любой современный браузер: Chrome, Firefox, Safari, Edge).
- **Планшет и смартфон** — адаптивная вёрстка: форма поиска, карточки текущей погоды и прогноза на неделю удобно читаются и на мобильном.
- **Среда** — фронтенд на Vercel, бэкенд на PythonAnywhere; работа через интернет, без необходимости запускать что-то локально на портах.

Ограничения: нужен доступ в интернет; для полной работы OpenWeatherMap в бэкенде задаётся API-ключ (опционально).

---

## Как работать (основные функции)

1. **Поиск по городу**  
   В поле «Город» ввести название (например: Минск, Берлин, Москва) и нажать кнопку — текущая погода и прогноз подставляются по координатам через геокодинг.

2. **Поиск по координатам**  
   В поля «Широта» и «Долгота» ввести значения (например для Минска: 53.9, 27.5) и запросить погоду — данные запрашиваются напрямую по этим координатам.

3. **Текущая погода**  
   После запроса отображаются карточки от каждого источника: температура, ветер (скорость на 10 м), при необходимости осадки; одна из карточек — DWD ICON.

4. **Прогноз на неделю**  
   Отдельный блок или кнопка «Прогноз на неделю»: по дням показываются макс/мин температура, ветер (максимум за день, порывы, преобладающее направление) на основе DWD ICON.

5. **Ветер по высотам**  
   В блоке прогноза или отдельной секции — таблица/карточки: высота (10 м, 80 м, 120 м, 180 м, ~110 м, ~320 м, … ~1000 м) и скорость/направление ветра (сейчас и/или по дням).

Технически: фронтенд ходит на бэкенд (`/api/weather`, `/api/forecast`); бэкенд обращается к Open-Meteo Geocoding, Open-Meteo Forecast, DWD ICON API и при наличии ключа — к OpenWeatherMap.

## Как выглядит страница (цвет и стиль)

- **Общий вид**  
  Одностраничный интерфейс: сверху заголовок, под ним форма поиска, ниже — блоки с результатами (текущая погода, прогноз на неделю, ветер по высотам).

- **Цвета**  
  - Фон страницы: тёмно-серый (`#1a1a1a`).  
  - Текст: светло-серый (`#e0e0e0`).  
  - Карточки (текущая погода, дни прогноза): фон `#2a2a2a`, рамка `#444`.  
  - Кнопки: акцентный сине-бирюзовый (`#0a7ea4`), при наведении темнее (`#096a8a`).  
  - Поля ввода: фон `#2a2a2a`, рамка `#444`.  
  - Ошибки и предупреждения: красноватый оттенок (например `#e57373`).  
  - Вспомогательный текст (подсказки, примечания): приглушённый серый (`#aaa`).

- **Типографика**  
  Системный шрифт: `system-ui, -apple-system, sans-serif`; заголовки и названия источников — без засечек, читаемо на любом экране.

- **Сетка и отступы**  
  Контент ограничен по ширине (например `max-width: 800px`), по центру. Карточки текущей погоды — сетка с автоматическим переносом (например `minmax(220px, 1fr)`). У блоков прогноза по дням и ветра по высотам — своя сетка или список с отступами и скруглёнными углами (`border-radius: 8px` у карточек, `4px` у полей и кнопок).

- **Адаптивность**  
  На узких экранах карточки и форма выстраиваются в один столбец, кнопки и поля остаются удобного размера для касаний.

Итог: тёмная, спокойная тема с синим акцентом на действиях; структура — «форма → текущая погода → прогноз → ветер по высотам».
